1. Docker + VcXsrv 运行 emacs-feature-igc 指南

1.1. VcXsrc 启动需要

启动 VcXsrv (XLaunch)：

在您的 Windows 宿主机上，启动 VcXsrv。

关键设置回顾：

  • Display settings: 选择 "Multiple windows"，Display number 设为 0。
  • Extra settings: 必须勾选 "Disable access control"。

  • Additional parameters for X Server: 添加 -xkboptions ctrl:nocaps（Docker环境: 没有硬件层级的caps 2 control的方法）

    并且在init.el最开始（或者能让此配置加载的el文件地方），添加上 

    (setq mac-command-modifier 'meta) ; 保持 Mac/Windows 上的 Meta 键行为
(setq mac-option-modifier 'alt)  ; 保持 Alt 键行为
(setq x-alt-keys-may-terminate-input nil) ; 避免 Alt/Meta 键的行为问题

;; 核心代码：将 Caps Lock 按键符号翻译为 Control 修饰键
(define-key key-translation-map (kbd "<caps>") (kbd "C-"))

    只能在 Emacs 上使用，因为这是 elisp神力

    虽然大写键的灯仍会亮起，但是在 Emacs 上，它统统翻译成了 Control ("C-")

    没有HHKB就这样咯，谁叫软软设计师键盘不塞一个硬件控制器进去呢？反正能用上Caps 2 Control就行了。

启动它并保持在后台运行。

1.2. Docker 启动需要

加载容器

当您在任何一台安装了 Docker 的新电脑上时，您只需要执行以下两个命令即可“复活”您的 Emacs 环境：

  1. 导入镜像 (Load)

将 .tar 文件导入到新的 Docker 环境中： 

# 语法：docker load -i [输入文件名]
docker load -i emacs-igc-horcrux.tar
结果： emacs-igc-horcrux:latest 镜像会出现在新电脑的镜像列表中。
  1. 运行容器 (Run)

像第一次启动一样，运行您的新镜像，并设置 X11 转发： 

docker run -it --name new-emacs-igc --hostname emacs-igc \
           -e DISPLAY=host.docker.internal:0.0 \
           emacs-igc-horcrux:latest \
           /bin/bash

结果： 一个全新的、包含您所有配置和编译成果的 Emacs IGC 环境就启动了！

通过这套流程，您可以确保您的复杂环境配置永远不会丢失，并可在任何地方快速部署。

1.3. 启动容器horcrux

1.3.1. 持久化装载到docker-desktop里（即使关闭powershell窗口仍可以运行此horcrux）

docker run -it `
  --name arch-emacs-igc `
  -e DISPLAY=host.docker.internal:0.0 `
  archlinux:base-devel

1.3.2. 进入容器horcrux

# 启动容器horcrux（约等于开机）
docker start arch-emacs-igc

# 进入容器horcrux（约等于进入桌面）
docker exec -it arch-emacs-igc /bin/bash

1.4. 重新打包容器horcrux

Q: OKOK这个我懂，但是，请问Gemini？既然docker的环境称为“容器”，那我能不能像伏地魔一样，打包这个容器，以后直接用上就可以了啊？

A: 这是一个绝妙的比喻！您想把您的 Emacs IGC 环境制作成一个 “魂器（Horcrux）”，在需要时随时召唤回来。

‍♂️ 制作 Emacs “魂器” (Docker Image) 答案是 完全可以，这也是 Docker 最核心的价值所在——创建可移植、可复现、带有所有配置的完整环境包。

您需要的操作分为两步：首先是“封印”当前运行的容器，然后是“打包”这个封印。

步骤一：封印容器 (Commit to an Image) 您当前的 arch-emacs-igc 容器包含了您所有的编译成果、依赖库和配置 (.bashrc, Emacs Lisp 配置等)。我们需要将这个运行中的容器的状态，保存为一个静态的 Docker 镜像 (Image)。

在您的 PowerShell 终端中执行（确保容器正在运行）： 

# 语法：docker commit [容器名称] [新镜像的名称:标签]
docker commit arch-emacs-igc emacs-igc-horcrux:latest

结果： 您的所有更改、安装和配置都被保存到了一个名为 emacs-igc-horcrux:latest 的新镜像中。

Vmmem 会在后台，进行创建一个新镜像

步骤二：打包镜像 (Save the Image) 现在，您需要将这个静态镜像导出成一个单一的、可移植的文件，这样您就可以把它拷贝到 U 盘或上传到云端。 

# 语法：docker save -o [输出文件名] [镜像名称]
docker save -o emacs-igc-horcrux.tar emacs-igc-horcrux:latest

结果： 在您执行该命令的目录下，会生成一个名为 emacs-igc-horcrux.tar 的文件。

恭喜！ 这个 .tar 文件就是您的 Emacs IGC 容器环境的完整“魂器”！

1.5. Emacs编译参数

下载emacs 

git clone https://github.com/emacs-mirror/emacs
git checkout feature/igc
chmod +x autogen.sh
./autogen.sh

编译参数 

./configure --sysconfdir=/etc --prefix=/usr --with-harfbuzz --with-modules --with-x-toolkit=gtk3 --with-mps --with-gnutls --with-rsvg --with-imagemagick --with-native-compilation

安装 

make -j4
make install
