1. 修改melpa源的telega包的telega-msg-heading的face的新版方法

1.1. 发生了什么问题？

我自己的一些face一直都不生效 

(with-eval-after-load 'telega
  (set-face-attribute 'telega-msg-heading nil :background "#ffffff" :extend nil))

上述无论有没有设置，都一直是这个效果: 一直都无法设置成 #ffffff 一直都是 #d9d9d9(gray85)

因为个人使用的一共有两套主题颜色分别，可能是我个人有一些 background 没设置成 dark 的原因吧(即两套主题颜色都是 :background light)，于是 telega-msg-heading 此 face 只能是 ~gray85~，在 我个人的配置的暗色主题下，非常瞎眼 因为 个人 实在受不了了于是打算去配置

高强度询问请教AI后只能确定是telega.el(新版)源码的原因

1.2. 解决方法

在 ~/.config/emacs/elpa/telega-20241118.1404/ 这里，修改 telega-cusomize.el 

cd ~/.config/emacs/elpa/telega-20241118.1404/
emacs -Q telega-cusomize.el

然后去掉两行属性 

;; 原作者zevlg写的
(defface telega-msg-heading
  '((((class color) (background light))
     :background "gray85" :extend t)
    (((class color) (background dark))
     :background "gray25" :extend t)     
    (t :inherit widget-single-line-field :extend t))
  "Face to display messages header."
  :group 'telega-faces)

;; 现在为了改成自定义的telega-msg-heading的face的颜色，所以改成
(defface telega-msg-heading
  '((((class color) (background light))
     :extend t)
    (((class color) (background dark))
     :extend t)     
    (t :inherit widget-single-line-field :extend t))
  "Face to display messages header."
  :group 'telega-faces)

;; 最后在当前文件下，M-x执行 byte-compile-file，就可以了
