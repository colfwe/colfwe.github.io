1. 在Linux上编译emacs-scratch-igc

1.1. 1. 基本介绍

1.1.1. (1) 什么是igc？

在 真-lisp 的编程语言，比如 scheme、common lisp、elisp 这些 lisp 方言，都是自带了一个 GC 垃圾回收机制的

对！如果您用过现代Cpp，还有Go等编程语言，您就一定清楚它们是自带了一个 GC 垃圾回收机制的。

然而在 mock-lisp(伪-lisp) 的编程语言下，就没有 GC 机制，比如Rust写的一个eww桌面组件，就是用的 yuck 这门伪lisp编程语言去编写结构的。

那么这样一来，我们的GNU Emacs用的是elisp，所以我们的GNU Emacs就天生就有了GC机制？

对啊！GNU Emacs它天生就有GC机制

然后有些佬，嫌这个GC机制控制得不够自由，于是就硬是用Cpp重新造了一个 新的GC机制 的 The (M)emory (P)ool (S)ystem 出来

于是这个 新的GC机制 就称呼为 (I)ncremental GC，增量GC，Delta GC的意思，简称I-G-C，所以最后简称igc

它用到的库文件，叫做 The (M)emory (P)ool (S)ystem 的 mps 所以一会要安装 libmps 就是这么来的

1.2. 2. 安装方法

BYD，检测 libmps.h 的 ./configure checking 折磨了我贼久

1.2.1. (1) 外部库文件libmps的安装

如果不想安装 homebrew 的话, 此仓库 https://github.com/kitevnn/dotfiles-archlinux/tree/main/res/libmps 里也有 2023年版的1.118.0版的Release版本

libmps .a 静态文件部分 🔧 libmps.a 🔧 libmps-debug.a .h 头文件部分 🔧 mpsacl.h 🔧 mpsavm.h 🔧 mpscamc.h 🔧 mpscams.h 🔧 mpscawl.h 🔧 mpsclo.h 🔧 mpscmfs.h 🔧 mpscmv2.h 🔧 mpscmvff.h 🔧 mpscmvt.h 🔧 mpscsnc.h 🔧 mps.h 🔧 mpsio.h 🔧 mpslib.h 🔧 mpstd.h 🔧 mpswin.h

首先安装 homebrew 在 Linux 上

然后 

# 安装2023年版的1.118.0版的mps
brew install libmps

# 安装最新版的mps
brew install libmps --HEAD

最后在 homebrew 的指定目录下，如果在Linux下全部默认的话，会在 /home/linuxbrew/.linuxbrew/Cellar 的 /libmps/1.118.0/ 下得到libmps的文件

libmps 📁 ./usr 📁 ./include

最后BYD，一定要 复制粘贴 到你自己的 系统级的/usr 与 系统级的/include

我试过无数种添加CFLAGS与LDLAGS的标志的方法，全部无效，它最后全部都会在./configure后提示说: "The libmps is missing"！

必须要复制到系统级的/usr/lib与系统级的/usr/include配置才能完成！

所以，复制到系统级的/us/libr与系统级的/include/include目录下 

sudo cp *.h /usr/include
sudo cp *.a /usr/lib

最后终于完成了 libmps 的安装

1.2.2. 2. 编译选项

点击直达: emacs-scratch-igc

先到 github仓库，并切换到 emacs-scratch-igc 分支

然后先运行 ./autogen.sh ，来生成 ./confiure 配置文件

再按照此编译选项设置

使用请看这里 

CFLAGS="-fmax-errors=1000" ./configure --sysconfdir=/etc --prefix=/usr --libexecdir=/usr/lib --with-tree-sitter --localstatedir=/var --disable-build-details --with-harfbuzz --with-libsystemd --with-modules --with-mps=yes 'CFLAGS=-march=x86-64 -mtune=generic -O2 -pipe -fno-plt -fexceptions -Wp,-D_FORTIFY_SOURCE=3 -Wformat -Werror=format-security -fstack-clash-protection -fcf-protection -fno-omit-frame-pointer -mno-omit-leaf-frame-pointer -g -ffile-prefix-map=/build/emacs/src=/usr/src/debug/emacs -flto=auto' 'LDFLAGS=-Wl,-O1 -Wl,--sort-common -Wl,--as-needed -Wl,-z,relro -Wl,-z,now -Wl,-z,pack-relative-relocs -flto=auto'

仅仅只是在 我自己这一生的编辑器只需要有GNUEmacs就足够了(20250123) 的基础上，添加了 --with-mps=yes 与 避免待会编译器编译时的错误提示CFLAGS 两个option而已（ 使用请看上面 参考请看下面） 

./configure --sysconfdir=/etc --prefix=/usr --libexecdir=/usr/lib --with-tree-sitter --localstatedir=/var --disable-build-details --with-harfbuzz --with-libsystemd --with-modules --with-x-toolkit=no --with-cairo 'CFLAGS=-march=x86-64 -mtune=generic -O2 -pipe -fno-plt -fexceptions         -Wp,-D_FORTIFY_SOURCE=3 -Wformat -Werror=format-security         -fstack-clash-protection -fcf-protection         -fno-omit-frame-pointer -mno-omit-leaf-frame-pointer -g -ffile-prefix-map=/build/emacs/src=/usr/src/debug/emacs -flto=auto' 'LDFLAGS=-Wl,-O1 -Wl,--sort-common -Wl,--as-needed -Wl,-z,relro -Wl,-z,now          -Wl,-z,pack-relative-relocs -flto=auto'

最后等待 ./confiure 配置完成就行了

1.2.3. 检验是否安装成功

进入GNU Emacs后，执行 M-x igc-stats 并在打开的新buffer上，按 wasd 的 s

如果能出现

IGCOBJBLV 854 40992 48 48 IGCOBJBUILTINSUBR 0 0 0 0 IGCOBJBUILTINSYMBOL 0 0 0 0 IGCOBJBUILTINTHREAD 0 0 0 0 IGCOBJBYTES 0 0 0 0 IGCOBJCONS 1876951 45046824 24 24 ……………………………………………………………………………

这样的提示文字，就能说明 mps 起效果了！

如果是 The Symbol igc-stats is void 的话，那么 一定是你自己在emacs-scratch-igc分支源码包下的./configure内，没有让它识别到libmps库文件！请你自己重新查看文档安装！

很抱歉我说得这么绝对，但必须要复制到系统级的/usr/lib与/usr/include，这是我碰壁无数次后得出的 [唯一] 最终解决做法，添加CFLAGS与LDFLAGS在我这里是没有任何效果的！
